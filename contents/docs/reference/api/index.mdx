### VariableSetSpec
VariableSetSpec defines the desired state of VariableSet.

A VariableSet is a reusable bundle of Terraform input variables that is
composed onto Workspaces by reference. Values can be literal strings or
drawn from a Kubernetes Secret or ConfigMap in the same namespace. The
workspace controller is what actually injects the resolved values into the
plan and apply Job pods as TF_VAR_\<name\> environment variables; this
controller is only responsible for validating references and surfacing
their status.




| Field | Description | Default | Validation |
| --- | --- | --- | --- |
| `description` _string_ | Description is a human-readable purpose for this VariableSet. Optional<br />and not consumed by any controller; it exists so that operators can<br />document intent in the same place as the data. |  | Optional: \{\} <br /> |
| `variables` _[Variable](#variable) array_ | Variables is the ordered list of Terraform variables this set<br />contributes. Each entry becomes TF_VAR_\<name\> on the plan and apply<br />pods of any Workspace that references this set. Order is preserved on<br />the wire so that consumers can rely on a stable iteration order when<br />computing hashes, but ordering does not change semantics within a<br />single set: duplicate names are rejected. |  | MinItems: 1 <br /> |
