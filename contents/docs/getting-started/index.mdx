Three pages take you from zero to a reconciling Terraform workspace:

- [Why Magos?](/docs/getting-started/why-magos) explains where Magos fits relative to existing Terraform delivery tools and the trade-offs the project deliberately makes.
- [Installation](/docs/getting-started/installation) walks through deploying Magos to a Kubernetes cluster using the official Helm chart.
- [Quickstart](/docs/getting-started/quickstart) creates a Project and a Workspace, then watches Magos drive the resulting Terraform module to applied state.

If you only have time for one page, start with the Quickstart. The Why and Installation pages are reference material you can return to once the moving parts feel familiar.
